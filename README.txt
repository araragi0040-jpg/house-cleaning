CleanFlow v006 ログイン・クラウド共有対応版

■ 今回の追加内容
・ログイン画面を追加
・管理者とスタッフの権限を分離
・スタッフは自分の担当案件だけを表示
・スタッフから管理画面へ移動できない構成
・画面上部に保存状態を表示
・Google Apps Script＋スプレッドシートへ接続可能
・案件、元請け、担当者、単価、請求書設定を複数端末で共有可能
・作業写真をGoogle Driveへ保存可能
・クラウド未設定時は、従来どおり端末内デモとして利用可能
・日付を固定値ではなく、端末の当日へ自動変更

■ まずデモを確認する方法
1. ZIPを展開します。
2. index.htmlをブラウザで開きます。
3. 次のアカウントでログインします。

管理者
  ID: admin
  PIN: 1234

スタッフ
  ID: yamada
  PIN: 1111

※ config.js の apiUrl が空欄の場合は、データはこの端末内だけに保存されます。

■ クラウド共有を有効にする方法
1. Googleスプレッドシートを新規作成します。
2. スプレッドシートの「拡張機能」→「Apps Script」を開きます。
3. 同梱の Code.gs の内容を貼り付けて保存します。
4. Apps Script上で setupCleanFlow() を1回実行します。
5. 権限確認が表示されたら承認します。
6. 「デプロイ」→「新しいデプロイ」→「ウェブアプリ」を選択します。
7. 次の設定でデプロイします。
   ・次のユーザーとして実行：自分
   ・アクセスできるユーザー：全員
8. 発行された /exec で終わるURLをコピーします。
9. config.js を開き、apiUrl の空欄へURLを貼り付けます。
10. GitHub／Vercel等へファイルを公開します。

■ GAS初期アカウント
管理者: admin / 1234
スタッフ: yamada / 1111
スタッフ: sato / 2222

重要：クラウド接続後は、Apps Scriptエディタで次のように実行し、初期PINを変更してください。
  setUserPin("admin", "新しいPIN")

利用者を追加・更新する場合：
  upsertUser("loginId", "表示名", "staff", "初期PIN", true)

※ role は管理者なら admin、スタッフなら staff を指定します。

■ 保存先
・案件：スプレッドシート Jobs シート
・元請け、担当者、単価：Config シート
・請求書設定：Config シート
・ログイン利用者：Users シート
・作業写真：Google Drive「CleanFlow_作業写真」フォルダ

■ 写真について
クラウドモードでは、登録写真をGoogle Driveへアップロードします。
初期版ではアプリから表示できるよう「リンクを知っている人が閲覧可能」で保存します。
本番運用前に、写真の保存方法・閲覧権限は事業者の運用ルールに合わせて再確認してください。

■ v005データを引き継ぐ場合
1. v005の設定画面からバックアップJSONを保存します。
2. v006へ管理者ログインします。
3. 設定画面からバックアップJSONを復元します。
4. クラウド接続済みの場合、復元した案件と設定が順次同期されます。

■ 主なファイル
・index.html：画面本体
・styles.css：デザイン
・app.js：画面・業務処理
・config.js：GAS URLなどの環境設定
・api.js：クラウド通信処理
・Code.gs：Google Apps Script側の処理
・manuals：現在の清掃マニュアルPDF

■ 現段階の位置づけ
v006は、複数端末共有へ進むための初期実装です。
正式運用前に、実際の元請け・担当者・単価・請求書情報へ置き換え、複数端末で同時操作した際の確認を行ってください。
