CleanFlow v007 GAS接続・複数端末テスト版

■ 今回の更新
・指定されたGASウェブアプリURLをconfig.jsへ反映
・クラウド共有を標準にし、端末内デモログインを停止
・ログイン時にクラウド読込へ成功しない場合、サンプルデータを表示しない安全な動作へ変更
・画面上部の同期表示を押すと、最新データを手動で再読込
・画面復帰時と約2分ごとに最新データを自動取得
・通信失敗時の変更内容を端末内に保持し、次回同期で再送
・写真登録前に案件データを先に同期し、Driveへの写真保存失敗を減らす
・クラウド運用時は「デモ初期化」ボタンを非表示

■ 確認する流れ
1. index.htmlを開くか、GitHub／Vercelへ公開します。
2. 管理者アカウントでログインします。
3. 設定画面で元請け・担当者・単価を確認します。
4. 管理者端末で案件を1件登録します。
5. 別ブラウザまたは別端末でスタッフログインし、案件が表示されるか確認します。
6. スタッフ側で写真・完了報告を登録します。
7. 管理者画面上部の同期表示を押し、作業報告が反映されるか確認します。

■ 注意
・Code.gsのAPI仕様はv006と互換のため、v006のCode.gsをすでにデプロイ済みなら差し替え不要です。
・GAS URLをブラウザで直接開くとdoGet未実装の表示になる場合がありますが、アプリはPOST通信を使用します。
・初期アカウントのPINをまだ変更していない場合は、Apps ScriptのsetUserPin()で変更してください。
・担当者名は、Usersシートの表示名とアプリ設定の担当者名を同じ表記にしてください。
・ローカルファイルからの通信がブラウザで制限される場合は、GitHub PagesまたはVercelへ公開して確認してください。
