CleanFlow v008 アカウント管理・PIN変更版

■ 今回の更新
・管理画面の「設定」からスタッフのログインアカウントを追加可能
・ログインID、表示名、権限、利用状態を管理可能
・管理者がスタッフのPINを再設定可能
・退職者などのアカウントを削除せず「停止」に変更可能
・担当者アカウントを作ると案件の担当者選択へ自動反映
・表示名を変更した場合、既存案件の担当者名も引き継ぎ
・管理者／スタッフ本人が右上の名前から自分のPINを変更可能
・右上の名前を押した際に「PIN変更」「ログアウト」を選択する形式へ変更
・CSS、JavaScriptにバージョン番号を付け、更新後のキャッシュ残りを軽減

■ 重要：GAS側の更新が必要です
v008ではログインアカウント管理用のAPIを追加しています。
現在のGAS URLはそのまま使用できますが、Code.gsの差し替えと再デプロイが必要です。

1. 使用中のスプレッドシートから「拡張機能 > Apps Script」を開く
2. 現在のCode.gsを、同梱のv008版 Code.gsへ全て差し替えて保存
3. 「デプロイ > デプロイを管理」を開く
4. 使用中のウェブアプリ右側の編集アイコンを押す
5. バージョンで「新バージョン」を選択
6. 「デプロイ」を押す

既存のスプレッドシート、案件、写真、GAS URLはそのまま引き継がれます。
setupCleanFlow()の再実行は不要です。

■ v008反映後の確認
1. 管理者でログイン
2. 「設定」を開く
3. 「ログインアカウント」に admin / yamada / sato が表示されることを確認
4. 初期アカウントを使用する場合はPINを変更
5. 実際のスタッフを「＋アカウント追加」から登録
6. 別ブラウザまたは別端末で新しいスタッフアカウントへログイン
7. そのスタッフへ割り当てた案件だけが表示されることを確認

■ PINについて
・4〜12桁の数字で設定します。
・管理者は設定画面から各スタッフのPINを再設定できます。
・本人は画面右上の自分の名前から変更できます。
・PINそのものはスプレッドシートへ保存せず、ハッシュ化された値だけを保存します。
